Abstract

With the VCR being abandoned for several years and with the tape lifetime almost reaching its limit, it is very useful for some user agent to be able to connect a VCR reader on their computer or smartphone and be able to read/record/stream their old VHDS tapes. This specification defines an API supporting the VHS protocol, enabling web applications to enumerate and select VCR input and output devices on the client system and send and receive VHS frames. It is intended to enable video applications as well by providing low-level access to the VCR devices available on the users' systems. At the same time, the Web VCR API is intended to become a semantic controller platform.

1. Introduction

This section is not normative.

This document introduces new HTML and CSS features to enable the use of video cassette. These are intended to bring back the joy of watching your old tapes, rewinding your cassette and enjoying the bleeding edge technology from a supporting User Agent.
