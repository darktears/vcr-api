Abstract

The Video Home System (better known by its abbreviation VHS) is a consumer-level analog recording videotape-based cassette standard. It is widely deployed and used these days by a big number of people. Unfortunately VHS's player were not designed to be connected to Internet, this specification was created to adress this issue : it is very useful for some user agent to be able to connect a VCR reader on their computer or smartphone and be able to read/record/stream their old VHS tapes. This specification defines an API supporting the VHS protocol, enabling web applications to enumerate and select VCR input and output devices on the client system and send and receive VHS frames. It is intended to enable video applications as well by providing low-level access to the VCR devices available on the users' systems. At the same time, the Web VCR API is intended to become a semantic controller platform allowing you to perfom operation to control the underlaying hardware.

The Tape Working Group discarded on purpose the Betamax protocol as its usage seems to be marginal.

1. Introduction

This section is not normative.

This document introduces new HTML and CSS features to enable the use of video cassette. These are intended to bring back the joy of watching your old tapes, rewinding your cassette and enjoying the bleeding edge technology from a supporting User Agent. The Web VHS API consists of the following set of interfaces and objects, as well as some supporting interfaces:

  • A NavigatorVHSAccess interface, which is supported on the window.navigator object in user agents. This interface has only one method which enables a developer to request access to VHS on the system.

2. Conformance

As well as sections marked as non-normative, all authoring guidelines, diagrams, examples, and notes in this specification are non-normative. Everything else in this specification is normative. The key words must, must not, required, should, should not, recommended, may, and optional in this specification are to be interpreted as described in [RFC2119]. This specification defines conformance criteria that apply to a single product: the user agent that implements the interfaces that it contains. Implementations that use ECMAScript to implement the APIs defined in this specification must implement them in a manner consistent with the ECMAScript Bindings defined in the Web IDL specification [WEBIDL], as this specification uses that specification and terminology.

3. Security and privacy considerations

There are security and privacy concerns with adding the Web VHS API to the web platform:
  1. Allowing the enumeration of the user's VHS interfaces is a potential target for fingerprinting.
  2. A malicious website could potentially send control the user's hardware and send malicious commands that could lead to potential loss of data.
These issues will be further explored prior to this specification becoming a Recommendation.

4. Obtaining Access to VHS Interfaces Available to the User Agent

IDL Definition 
		  [NoInterfaceObject]
		  interface NavigatorVHSAccess {
		   bool getVHSAccess();
		  };

Methods

getVHSAccess
Obtains an interface to enumerate and request access to VHS devices on the user's system. Returns true if there is a VCR connected to the User Agent, false otherwise. This method is a blocking and User Agent is required to block any other script execution (such a JS Workers for example).
Note : This method does not have a callback for error handling, we expect that querying VHS devices is always a successful operation. After so many years on the market, VCR are very stable and robust hardware.

5. VHSAccess Interface

IDL Definition 
		  [NoInterfaceObject]
		  interface VHSAccess {
		   VHSInput getInput(short usb_port);
		   VHSOuput getOuput(short usb_port);
		  };

Methods

getInput
Obtains an VHSInput which you can manipulate to perform input (or commonly named write) operations on a VCR.
Property :
usb_port of type short, readonly
The USB port number of which the VCR is connected. Passing -1 will perform a search on all USB ports to find the first available VCR.
getOuput
Obtains an VHSOutput which you can manipulate to perform output (or commonly named read) operations on a VCR.
Property :
usb_port of type short, readonly
The USB port number of which the VCR is connected. Passing -1 will perform a search on all USB ports to find the first available VCR.
Note : User is expected to use VHSInput and VHSOuput in concert to control the VCR.

6. VHSInput Interface

IDL Definition 
		  [NoInterfaceObject]
		  interface VHSInput {
		   void rewind();
		   void forward();
		   void pause();
		   void play();
		   void eject();

Methods

rewind
Rewind the current tape inside the VCR. VCRs are unable to tell the length of the tape and the current progress. There is therefore no information on the current progress of rewinding the tape.
forward
Forward the current tape inside the VCR. VCRs are unable to tell the length of the tape and the current progress. There is therefore no information on the current progress of forwarding the tape.
play
Launch playback of the current tape inside the VCR.
pause
Pause the playback of the current tape inside the VCR.
eject
Eject the current tape inside the VCR. The execution of this method is expected to be around 2-3 seconds.
Note : rewind(), forward(), pause(), play() are synchronous methods, the script execution is expected to be blocked.

7. VHSOuput Interface

IDL Definition 
		  [NoInterfaceObject]
		  interface VHSOutput {
		   void streamToYouTube(DOMString login, DOMString password);
		   void setVideoOutput(HTMLElement* element);
		   void setUpscaleVideoOutput(DOMString quality);

Methods

streamToYouTube
Stream the current tape to YouTube. User are expecting to launch the playback previously with VHSInput interface.
Property :
login of type DOMString, readonly
The login of the user.
password of type DOMString, readonly
The password of the user.

Note : Two steps logins are not supported by this level.
setVideoOutput
Set the video output on the element passed in parameter. The element should be a video tag to be able to render the video. If a audio element is passed only the sound of the tape will be played.
Property :
element of type HTMLElement, readonly
The element on which the VHS should be played, video element for both audio and video support, audio element for audio only playback.
setUpscaleVideoOutput
Set how the video should be upscaled, note that native VHS resolution is 333x480.
Property :
quality of type DOMString, readonly

The value of the video output quality. Possible values are :
  • 480i
  • 576i
  • 480p
  • 576p
  • 720p
  • 1080i
  • 1080p
  • 2160p
  • 4320p
  • 8640p
The final quality is at the discretion of the User Agent and how upscaling is implemented.
